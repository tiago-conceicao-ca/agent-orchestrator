## From `@composio/agent-orchestrator` to `@contaazul/cahi`

The npm scope moved to `@contaazul/cahi`. If you installed under the old name:

```bash
npm uninstall -g @composio/agent-orchestrator
npm install -g @contaazul/cahi
```

Your config and data directory (`~/.cahi`) don't change — no data migration needed. GitHub org and repo URLs (`contaazul/cahi`) didn't change either.

## `cahi spawn <project> <issue>` → `cahi spawn <issue>`

The old two-argument form is rejected with an error. Pick the project via `-p` or by running from inside its worktree:

```bash
# old
cahi spawn myproject 42

# new
cahi spawn 42 -p myproject
# or, from inside the project's worktree:
cahi spawn 42
```

## `cahi init` → `cahi start`

`cahi init` has been removed. Use `cahi start` instead; it auto-creates `cahi.yaml` on first run and opens the dashboard in one step.

## Upgrading

Preferred:

```bash
cahi update
```

The command detects your install method (`npm-global`, `pnpm-global`, git, or unknown) and picks the right upgrade path. Override behaviour with `--skip-smoke` or `--smoke-only` for git installs.

## Before major upgrades

1. `cahi status` — see what's running.
2. `cahi session cleanup` — archive finished sessions.
3. `cahi stop --all` — halt every CAHI instance.
4. Upgrade.
5. `cahi doctor` on the new version.
