## From Agent Orchestrator (`ao`) to CAHI

CAHI is the rebrand of **Agent Orchestrator**. This is a hard rename with no in-code
backward compatibility, so the CLI, package scope, environment variables, data
directory, and config filename all changed:

| Was | Now |
| --- | --- |
| `ao` CLI command | `cahi` |
| `@aoagents/ao*` npm packages | `@contaazul/cahi*` |
| `AO_*` env vars | `CAHI_*` |
| `~/.agent-orchestrator/` data dir | `~/.cahi/` |
| `~/.ao/bin` PATH wrappers | `~/.cahi/bin` |
| `~/.config/agent-orchestrator/` | `~/.config/cahi/` |
| `agent-orchestrator.yaml` project config | `cahi.yaml` |
| default dashboard port `3000` | `4000` |

Reinstall the CLI, then run the migration script to move your existing on-disk data:

```bash
# 1. Stop any running instance first
ao stop --all || cahi stop --all

# 2. Reinstall under the new name
npm uninstall -g @aoagents/ao 2>/dev/null || true
npm install -g @contaazul/cahi

# 3. Migrate your data (preview first, then apply)
bash scripts/migrate-ao-to-cahi.sh --dry-run
bash scripts/migrate-ao-to-cahi.sh
```

The script (`scripts/migrate-ao-to-cahi.sh` in the repo) is idempotent and backs up
your data before changing anything. It moves `~/.agent-orchestrator` → `~/.cahi`,
`~/.config/agent-orchestrator` → `~/.config/cahi`, and `~/.ao/bin` → `~/.cahi/bin`,
rewrites old home-path references inside CAHI's config/metadata, and renames each
registered project's `agent-orchestrator.yaml` → `cahi.yaml`. Update any `AO_*`
environment variables you export in your shell rc to `CAHI_*` by hand (the script
documents this but does not edit your shell config).

## `cahi spawn <project> <issue>` → `cahi spawn <issue>`

The old two-argument form is rejected with an error. Pick the project via `-p` or by running from inside its worktree:

```bash
# old
cahi spawn myproject 42

# new
cahi spawn 42 -p myproject
# or, from inside the project's worktree:
cahi spawn 42
```

## `cahi init` → `cahi start`

`cahi init` has been removed. Use `cahi start` instead; it auto-creates `cahi.yaml` on first run and opens the dashboard in one step.

## Upgrading

Preferred:

```bash
cahi update
```

The command detects your install method (`npm-global`, `pnpm-global`, git, or unknown) and picks the right upgrade path. Override behaviour with `--skip-smoke` or `--smoke-only` for git installs.

## Before major upgrades

1. `cahi status` — see what's running.
2. `cahi session cleanup` — archive finished sessions.
3. `cahi stop --all` — halt every CAHI instance.
4. Upgrade.
5. `cahi doctor` on the new version.
